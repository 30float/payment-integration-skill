# Roadmap

## Now

Current V1 focus:
- Stripe-only
- recurring subscriptions only
- Next.js as the first-class implementation target
- user-based and workspace-based billing
- Stripe Checkout + Customer Portal
- webhook-driven billing truth and app-side entitlement sync
- plan-first workflow before implementation

## Next

Likely next expansions after V1 is battle-tested:
- stronger audit mode guidance
- clearer migration guidance for partial Stripe implementations
- one-time payment support
- seat-based billing guidance
- more robust examples across supported Next.js stacks

## Later

Potential future directions:
- LemonSqueezy
- Paddle
- usage / credits / metered billing
- broader framework support beyond Next.js
- more migration and normalization playbooks

## Not planned for V1

Not first-class for the initial release:
- Stripe Connect
- marketplace payouts
- split payments
- mobile-native subscription flows
- fully custom embedded card-entry flows as the default path
- enterprise invoice-heavy billing workflows
